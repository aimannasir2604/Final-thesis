# Contributing Guide

اگر دوسرے لوگ اس thesis پر کام کریں تو یہ rules ہیں:

## Branch Naming
- `main` - Production branch
- `develop` - Development branch
- `feature/xyz` - نیا feature
- `fix/xyz` - Bug fix

## Commit Messages (اردو میں)
```
[TYPE] مختصر تفصیل

TYPE:
- feat: نیا feature
- fix: Bug fix
- docs: Documentation
- refactor: Code refactor
- test: Test cases
```

## Pull Request Process
1. اپنا branch بنائیں
2. Changes کریں
3. Commit کریں
4. Push کریں
5. Pull Request بھیجیں
6. Review کے بعد Merge کریں

## Tips
- ہمیشہ meaningful commit messages لکھیں
- Related files ایک ساتھ commit کریں
- بڑے changes میں چھوٹے commits کریں
